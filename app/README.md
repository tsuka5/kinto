# Kinto — 自己実験Webアプリ（N=1）

「努力ゼロで、安く、脳のパフォーマンスを上げる」という思想を、自分自身で検証するためのPWA。
**データはすべて端末内（ブラウザのlocalStorage）にのみ保存され、外部には一切送信されません。**

## 機能
- **提案カテゴリ（今日の一手の最上部に常設）** … 画面上部のチップで、提案してほしいジャンルを**単一選択**（🍖メイン／🥦野菜／🍚主食／🧀乳製品）。同じものを再タップで解除。その場で提案が変わる。
- **栄養コスパランキング（専用タブ）** … カテゴリ別に「金額あたりの栄養素（切替可）」が多い順にランキング表示。**各食材をタップ → 栄養データ（PFC・その他を1食目安と比較）→「料理を見る」→ レシピ → クックパッドの類似レシピへ遷移**、という導線をモーダルで提供。
- **店頭で相場チェック（🏷️「安い？」タブ）** … スーパーで値札を見た瞬間に使う専用ページ。食材を検索（かな漢字OK）→ 値札の「値段」「内容量」を大きな入力欄に打つと、**打った瞬間に「🟢安い！買い／⚪相場どおり／🔴高い」をデカ文字で即判定**（相場比%・¥/100g換算・あなたの過去最安も表示）。そのまま「これに決定（記録）」「価格だけ記録」「料理を見る」へ直結。※店頭判定はこのタブに一本化（食材詳細モーダルからは廃止）。
- **クックパッドリンク** … 料理カード（この一手で作るなら）とモーダルの料理ヘッダーに、レシピ名・主材料名での検索リンクを設置。
- **採用の取り消し** … 今日の一手を採用した後、「↩ 取り消す」で取り消し可能。
- **今日の一手（能動的に選ぶフロー）** … ①提案カテゴリを選ぶ（または**食材名で検索**。全カテゴリ横断・**ひらがな/カタカナ/漢字どれでもヒット**・選ぶとカテゴリ自動切替） → ②「今日買おうと思っているもの」を選ぶ → ③**それが最適なら「✅ それが最適です」**、もっと安く栄養がとれるなら**候補をランキング表示**（食材を選ぶと、**食材詳細と同じ「1食目安バー」形式の栄養カード**（カロリー＋PFC＋オメガ3・カルシウム・鉄・ビタミンD。**食材に合った自然な単位**（卵は1個・納豆は1パック・食パンは1枚など。単位がないものは100g）あたりの量を「1食でとりたい目安量」と並べて表示）と「◯◯にする（決定）」ボタンが先に表示され、その下のランキングへ自動スクロール。各候補は「栄養マッチ◯%」を**バーで視覚表示**＋相場価格（¥→¥）と¥◯おトク。**最安級の食材を選んだ場合も「👑 最安級」判定とともに栄養が近い類似食材を必ず提案**（飽き・売り切れ対策。相場が高い候補は×倍率表示））。各候補は「詳細」で栄養確認、「これにする」で乗り換え購入を記録。「そのまま買う」も可。「他のカテゴリも見る」で対象拡張。
- **料理提案＋栄養可視化＋1食コスト** … 一手を採用すると、その食材を使った簡単料理を提案。料理全体の栄養を「PFC」「その他」に分けて**1食の推奨量と比較**、さらに**1食あたりのコスト（円）**も表示。
- **不足の補完提案（ユーザー主導）** … 料理が1食の目安に届かない栄養を提示。**料理の栄養を見たうえで、優先的に補いたい栄養をユーザーが選択**でき、安く足せる食材を提案。
- ※「今日の一手」では、節約額と栄養の近さのみ表示（手間の指標は廃止）。栄養アシストカードも廃止。
- **食べ合わせの良いもう一皿＋追加後の合計栄養** … クックパッドリンクの後に、不足栄養を補い、かつ**元の料理と①献立の役割（主菜↔副菜/主食）②時間帯（朝/昼/夜）が合う一皿**を提案。さらに**追加後の合計栄養（元＋もう一皿 vs 1食目安）**をPFC・その他で表示。
- **レシピ（101品・全食材2品以上・朝昼夜タグ・日替わり）** … 各食材に複数レシピを用意し、日替わりでローテーション。全品に**番号付きの手順（火加減・レンジW数・時間入り）／所要時間（⏱）／💡コツ（調理テク・栄養の根拠）**を収録。料理名に「朝食・昼食向き」を表示。
- **栄養マッチ（方向×量・カテゴリ別軸）** … ①各栄養素を「100kcalあたり」に正規化し、食品中の最大値で割って単位・桁差を除去 → コサイン類似度で栄養バランスの「方向」の近さを算出。②さらに、元食品が1食目安に対して意味のある量を持つ栄養素について、候補が量をどれだけカバーするか（カバー率）を掛け合わせる。方向だけのコサインでは「量が1/4でも100%」になる見当外れを排除。**比較に使う栄養軸はカテゴリで切替**：メイン=たんぱく質・オメガ3・鉄・ビタミンD・Ca／**野菜=ビタミンC・ビタミンA・食物繊維・葉酸・鉄・Ca**／主食=炭水化物・食物繊維・たんぱく質／乳製品=Ca・たんぱく質・ビタミンD。食材詳細の「その他の栄養素」バーとコスパタブの選択肢も同じ軸に連動（ビタミン・繊維の1日目安は内部固定値：食物繊維21g・ビタミンC100mg・ビタミンA850µg・葉酸240µg）。
- **値動き（📉タブ）** … 最近**値下がりした食材**（今が買いどき）と**値上がりした食材**（タップで詳細から安い代替を探せる）を一覧表示。e-Stat実データがあれば前回データ比の実値、なければ暫定のデモ値＋e-Stat連携への誘導を表示。（旧・実績ダッシュボードはこのタブに置き換え）
- **マイ価格メモ** … 相場判定に入力した実売価格を自動記録。食材詳細に「あなたの最安・直近価格」を表示し、自分専用の価格DBに育つ。
- **相場の自動更新（e-Stat実データ・設定不要）** … アプリ内蔵の共有キーで、政府統計「小売物価統計調査」APIから**今月の実売価格を月替わりで自動取得**（23品目・手動更新ボタンもあり）。地域はIP判定した都道府県の県庁所在市（東京は「特別区部」）を使用。取得後は相場・ランキング・値動き・料理コストすべてが実データ基準になる。自分のappIdを設定すると共有キーより優先。
- **電光掲示板** … 相場を更新すると、**今月安くなった食材（▼◯%）**がヘッダー下をLED掲示板風に流れる。タップで食材詳細へ、触ると一時停止。
- **AIレシピ検索（✨ Claude API）** … 設定タブにClaude APIキーを登録すると、決定後の画面とレシピモーダルから**ClaudeがWeb検索で「最近人気のレシピ」を探して端末にためられる**（1日3回まで）。ためたレシピは一覧からその日の気分で選べ、手順・コツ・参考URL付き。キーは端末内にのみ保存し、api.anthropic.com 以外へは送信しない（利用料はキー所有者に課金・1回数円〜数十円目安）。ブラウザから直接 Claude API（`claude-opus-4-8`＋`web_search`ツール、CORSは `anthropic-dangerous-direct-browser-access` ヘッダーで許可）を呼ぶ構成。
- **食材詳細からの決定** … 詳細モーダルに「🛒 ◯◯にする（決定）」ボタン。コスパ表・電光掲示板・類似食材など、どこから開いても即決定できる（決定後は「◯◯に決定！」画面へ自動遷移。単価計算UIは「安い？」タブに一本化）。
- **1日の目標とPFCバランス（🍽 設定タブ・カロミル方式）** … ①**1日の目標カロリー** → ②そのカロリーをとる**PFCバランス（%でもgでも入力可。片方を入れるともう片方が自動計算、2つ入力すると残り1つは合計100%になるよう自動補完）** → ③**1日の食数（1〜6食）** の順に設定。グラム換算（P=kcal×比率÷4、F=÷9、C=÷4）と**1食の目安（＝1日の目安÷食数）**を自動算出し、栄養バー・ランキング・不足判定すべてに反映。**その他の栄養素（オメガ3・カルシウム・鉄・ビタミンD）は目標カロリーに比例して自動計算**（入力中にリアルタイムプレビュー・合計100%チェック付き）。設定には**名前を付けて保存**（例：減量期／増量期）でき、一覧から**ワンタップでいつでも過去の設定に戻せる**（食数ごと復元）。適用中の設定は再起動後も維持。
- **戻る機能** … 画面**左上に常駐する半透明の「←」ボタン**（スクロールしても追従。戻り先がないときはさらに薄く・無効化）。**スマホの戻るボタン／ブラウザの戻る**にも対応（モーダルが開いていればまず閉じる→次に前のタブへ、の順）。
- **API設定へのジャンプ** … 「API設定を開いてキーを登録する」ボタンやコスパタブの「e-Stat連携すると実値化」リンクから、設定タブの該当カードへ直接スクロール＆入力欄フォーカス。
- **はじめてのチュートリアル** … 初回起動時に6ステップのガイドを自動表示（ようこそ→今日の一手→安い？→コスパ→値動き→設定）。**説明中のタブが実際に背後に表示されるライブツアー形式**で、スキップ可・「設定 → 📖 チュートリアルを見る」でいつでも再表示できる。
- **全画面表示** … 画面**右上に常駐する「⛶」ボタン**でブラウザのURLバーを消してフル表示（もう一度押すと解除）。非対応ブラウザ（iPhoneのSafari等）では「ホーム画面に追加」の案内を表示。インストール済みPWAではもともと全画面のためボタンは非表示。ページは常にスクロール可能にしてあり、スクロールでもURLバーが自動で隠れる。

## 使い方（ローカル）
- いちばん簡単：`index.html` をダブルクリックしてブラウザで開く（オフラインで動作）。
- 初回は画面上部の**提案カテゴリ**チップを選ぶ。すると「今日の一手」が提案されます。
- ※ ファイルを直接開く方式（file://）ではPWA（ホーム画面追加）は無効。下記デプロイ後に有効になります。

## スマホで毎日使う（推奨：Vercelで公開）
**重要：公開しても健康データは出ません。** 公開されるのはアプリのコードだけで、計測データはあなたのスマホのブラウザ内に保存されます。

### 手順（GitHub + Vercel）
1. この `app/` フォルダだけを、**Private** のGitHubリポジトリにpush（戦略ドキュメントとはリポジトリを分ける）。
2. [vercel.com](https://vercel.com) にGitHubでログイン → "Add New Project" → 当該リポジトリを選択。
3. Framework Preset は **"Other"**（ビルド不要の静的サイト）。Root Directory は `app`（リポジトリ直下にappだけ置いた場合は不要）。
4. Deploy → 発行されたURLをスマホで開く。
5. スマホのブラウザメニューから **「ホーム画面に追加」** → アプリのように毎日起動できる（PWA）。

### 代替デプロイ先
- **Cloudflare Pages / Netlify** … Vercelとほぼ同等。静的サイトとしてそのまま公開可。
- **GitHub Pages** … 最小構成。Settings → Pages で公開。

## 自己実験のコツ（N=1）
- まず提案カテゴリを選び、毎日「今日の一手」を見て採用/スキップする習慣をつくる。
- 「値動き」で最近値下がり・値上がりした食材をチェックして買いどきを判断する。
- 体調・睡眠・繁忙など交絡要因が多いので、N=1は「示唆」レベル。それでも自分の手応えを掴むには十分。

## 食品データ
- 内蔵食材は**49品**（卵・肉10（皮なし/皮つきの鶏むね・もも、豚こま/豚バラ/合いびき/ささみ含む）・大豆3・魚6・乳製品3・野菜20（なす/きゅうり/レタス/じゃがいも/長ねぎ含む）・主食6＋レシピ用の調味料等）。栄養は文科省 食品成分表の代表値。**各カテゴリ内は料理でよく使われる順**に並び、ピッカー・「安い？」では**サブカテゴリの小見出し**（🥚卵・大豆／🍖肉／🐟魚・缶詰、野菜は🧺定番・🍅実もの・🥬葉物・🍄きのこ）でグループ表示。
- 配信時（Vercel等）は**文科省 食品成分DB（2,478食品）= `foods.full.json`** を読み込み、内蔵食材の栄養を八訂の公式値に自動補正します（`../poc/buildFoodDb.js` で生成。file:// で開いた場合は内蔵値で動作）。

## デザイン
- ライトなガラス基調（グラスモーフィズム）＋ティール→インディゴの動くグラデーション。背景にはゆっくり漂うオーロラ、カードは出現アニメ、栄養バーは0から伸びるアニメ付き。配色は CSS の `:root` 変数（`--grad` ほか）で一括管理。
- OSの「視差効果を減らす」設定（prefers-reduced-motion）ではアニメーションを全て無効化。

## e-Stat 連携（相場データの自動更新）
- **アプリ内で完結**：設定タブ「📡 相場データの自動更新」に無料の appId を貼ると、e-Stat「小売物価統計調査（動向編）」APIから**ブラウザが直接**今月の実売価格を取得（APIはCORS対応を確認済み）。
- 手順：① https://www.e-stat.go.jp/api/ でユーザー登録（無料・数分）→ appId 発行 → ② 設定タブに貼り付け → ③「今月の相場に更新」。以後は**月が変わるとアプリ起動時に自動更新**。
- **共有キー方式**：`index.html` の `ESTAT_SHARED_APP_ID` に開発者の appId を1つ入れてデプロイすると、**利用者は各自で取得しなくても自動更新が使える**（利用者が自分のキーを入力した場合はそちらが優先）。appId はブラウザのリクエストに載る公開情報のため隠せない点、レート消費・利用責任が登録者に集約される点に留意。大規模公開時は e-Stat の利用規約確認を推奨。
- 地域：IPで判定した都道府県の**県庁所在市**（例：愛知県→名古屋市）の価格を取得。手動で都道府県を変えた場合もそれに追従。
- 品目対応：牛・豚・鶏・さけ・まぐろ・鶏卵・牛乳・食パン・米・豆腐・ほうれん草・ブロッコリー等を¥/100gに換算（単位換算は目安。異常値は自動除外）。未対応食材は内蔵の目安相場のまま。
- 旧 `../poc/estatClient.js`（Node版）も検証用に残置。

## 本番への発展
- 食品/価格データ → e-Stat API・文科省 食品成分DB・EC連携の実データに置換（`../poc/` 参照）。
- 根拠文 → PubMed要約をClaudeで自然文化（`../poc/pubmedClient.js` 参照）。
- 複数人実験（Phase 0でN>1）→ Supabase等の無料バックエンドでクラウド保存。
