# Kinto PoC — 代替品マッチングエンジン / PubMed連携

第2ステップで確立したアプリ「Kinto」のコア機能（栄養 × コスト × 手間ゼロの"格上の代替"提案）の技術検証用PoC。

## 構成
- `foods.sample.json` … 文科省『日本食品標準成分表』の代表値を参考にしたサンプル食品データ（可食部100gあたり）。price/effortはPoC用ダミー。
- `matchEngine.js` … 代替提案エンジン本体。栄養効率ベクトルのコサイン類似度 × 価格メリット × 手間メリットで候補をランク付けし、誠実な根拠文を生成する。
- `pubmedClient.js` … PubMed E-utilities APIクライアント（無料）。代替提案の科学的根拠となる論文メタデータを取得する。

## 実行方法
```bash
# 代替提案（既定: サーモン/ほうれん草/牛ロース）
node matchEngine.js

# 特定食品の代替提案（idはfoods.sample.json参照）
node matchEngine.js salmon_raw

# PubMedで根拠論文を検索（要ネットワーク。NCBI_API_KEY環境変数があればレート上限UP）
node pubmedClient.js "frozen vegetables nutrient retention"
```

## マッチングロジック（要点）
1. 各食品の主要栄養素（たんぱく質・オメガ3・カルシウム・鉄・ビタミンD）を **100kcalあたり** に正規化 → "栄養の質"ベクトル化。
2. 元食品とのコサイン類似度で「栄養プロファイルが近い」候補を抽出。
3. `score = 栄養類似度 × (1 + 0.8×価格メリット + 0.4×手間メリット)` で総合評価。
4. 上位を「節約額・栄養差分・誠実な根拠文」付きで返す。

## 本番への発展（Phase 0→）
- `foods.sample.json` → 文科省 食品成分DB API ＋ e-Stat 小売物価統計 ／ EC公式API・アフィリエイト連携に置換。
- 根拠文のルールベース生成 → 取得したPubMed abstractを **Claude** で要約し自然文化。
- スコア重み（0.8 / 0.4）はA/Bテストと実データで最適化。

## 留意点
- 価格・手間はダミー値。栄養値も代表値であり、本番では正確な成分表データに差し替える。
- スクレイピングを行う場合は著作権法30条の4の範囲・各サイト利用規約・低負荷設計を順守（岡崎市立中央図書館事件を踏まえる）。
